# Hi; I'm Benjie!

#### Need help? [Book a call!](/book)

I'm a [community funded](https://github.com/sponsors/benjie) open-source
developer, working in the intersections of GraphQL, Node.js and PostgreSQL
under the umbrella "Graphile".

Regarding GraphQL; I'm a [member of the
TSC](https://github.com/graphql/graphql-wg/blob/main/GraphQL-TSC.md#tsc-members)
(technical steering committee), a significant [contributor to the
spec](https://github.com/graphql/graphql-spec/graphs/contributors), and am very
[active in the working
group](https://github.com/graphql/graphql-wg/graphs/contributors) community.
Looking to contribute to the GraphQL specification, but need some guidance?
Reach out to me via [the GraphQL Discord](https://discord.graphql.org) in the
`#graphql-spec` channel — I'm @Benjie — and I'll help however I can.

If your company uses (or is considering using) any of the software in the
Graphile stack (PostGraphile, Gra*fast*, Graphile Worker, Graphile Migrate,
Graphile Starter, pgRITA, etc) and wishes to consult with me, you can [book a
call now](/book), or reach to my first name @graphile.com to discuss other
options.

## Maintainer

Some of the open source projects I maintain:

- [Gra*fast*](https://grafast.org) - a cutting-edge planning and execution
  engine for GraphQL
- [PostGraphile](https://postgraphile.org) - an unopinionated tool to speed up
  development of extremely high performance GraphQL APIs backed primarily by a
  PostgreSQL database
- [Graphile Worker](https://github.com/graphile/worker) - a high-performance
  Postgres-backed job queue
- [Graphile Migrate](https://github.com/graphile/migrate) - a highly opinionated
  forward-only migration framework for PostgreSQL
- [Graphile Starter](https://github.com/graphile/starter) - a highly opinionated
  SaaS starter backed by the Graphile suite

Find more on [my GitHub profile](https://github.com/benjie).
